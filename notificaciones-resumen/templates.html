Cómo se muestra el gráfico (handoff a ingeniería)

En el prototipo el gráfico es un SVG porque corre en el navegador. En un correo o mensaje real no se usa SVG ni JavaScript (los clientes los bloquean): el gráfico se muestra como una imagen PNG renderizada en el servidor al momento de armar el resumen.

  1. Render (backend): por cada incidente se "fotografía" su gráfico en la ventana de tiempo correspondiente. Opción fiel: headless browser (Playwright) capturando el componente real del tablero. Opción liviana: librería de charts del lado del servidor (ECharts SSR, Vega). Renderizar a 2x y mostrar a la mitad del ancho para que se vea nítido.
  2. Email: embeber con <img> mediante adjunto CID (src="cid:chart-incidente-123") o una URL firmada con expiración corta. Nunca un bucket público: es data del cliente.
  3. Slack: usar un bloque image de Block Kit apuntando al mismo PNG hospedado. El render se hace una sola vez y sirve para los dos canales.
  4. Diseño y accesibilidad: la imagen es estática, sin hover ni interacción. Siempre con alt descriptivo del incidente (muchos clientes bloquean imágenes por defecto). La narrativa de texto debe entenderse aunque la imagen no cargue.
